# Router testing (quando solicitado)

## Regra local

- Evitar mock de Router; use harness quando precisar testar navegação.
- Não validar HTML; foque no estado e métodos.
- Para testes unitários de lógica fora do fluxo de navegação, siga `vitest-unit-tests`.

## Nota

- Router testing complementa os unit tests e não substitui a política principal da skill de testes unitários.
